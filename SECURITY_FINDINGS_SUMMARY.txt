3. CRITICAL: Frontend Token Storage Not Encrypted
   Location: /Users/usangaraju/Downloads/smartfarm/mobile/src/store/useAuthStore.js
   Lines: 8, 16-45, 55
   CVSS Score: 8.7

   Issue: JWT tokens stored in plain-text AsyncStorage
   - No encryption on disk
   - Readable by any app with file access
   - Extracted from device backups

   Attack Vector: Physical device access, malware, unencrypted backups

   Fix Timeline: 3-4 hours
   Action Items:
   a) npm install react-native-encrypted-storage
   b) Replace AsyncStorage with EncryptedStorage for token storage
   c) Test token persistence across app restarts
   d) For web: Use memory-only storage (cleared on page refresh)

4. CRITICAL: Missing Authorization Checks (IDOR - Insecure Direct Object References)
   Location: Multiple routers
   CVSS Score: 8.4

   Vulnerable Endpoints:
   - /api/service-requests/{request_id} (GET) - No ownership check
   - /api/supply-chain/transfers/{transfer_id} - No ownership check
   - Estimated 40+ endpoints without ownership/role validation

   Attack Scenario:
   User A (unprivileged) requests: GET /api/service-requests/999
   Response: Full details of another user's sensitive service request

   Fix Timeline: 3-5 days
   Action Items:
   a) Audit all 175+ endpoints using db.query().filter() patterns
   b) Add ownership checks: if resource.user_id != current_user.id: raise 403
   c) Add role-based checks for admin operations
   d) Implement ownership validation helper function
   e) Write tests for unauthorized access scenarios

   Location: /Users/usangaraju/Downloads/smartfarm/backend/routers/auth.py
   Lines: 32-49, 163-189
   CVSS Score: 8.3

   Issues:
   a) In-memory rate limiting lost on server restart
   b) No persistence across multiple server instances (horizontal scaling)
   c) Can be bypassed using different IP addresses (proxy/VPN)
   d) Only 5 attempts before 15-minute lockout (too lenient)

   Fix Timeline: 2-3 hours

   b) Implement persistent rate limiting with Redis
